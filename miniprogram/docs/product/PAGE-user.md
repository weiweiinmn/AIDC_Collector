## 2. 页面定位与功能概述

用户管理模块用于管理 AIDC Collector 管理后台的团队成员账号，包括账号的创建、角色分配、状态启停和权限配置。所有人员必须经由管理员统一分配账号后方可登录系统。

---

## 3. 用户角色与权限

| 角色 | 可见页面 | 权限说明 |
|------|---------|---------|
| admin | 全部（含管理员设置） | 系统最高权限，可管理所有用户 |
| manager | 仪表盘、数据管理页、团队成员（只读） | 可管理数据，无法操作用户 |
| viewer | 仪表盘、机房/客户（只读） | 仅可查看，无写权限 |

> 普通用户（manager/viewer）访问用户管理页面时，只能看到成员列表，无新建/编辑/删除按钮。

## 6. 交互流程

### 6.1 新建成员（admin）

```
点击「新建成员」→ 弹出新建弹窗/跳转新建页
  → 填写：姓名、用户名、手机号、邮箱、角色、部门、初始密码
  → 实时校验用户名唯一性（失焦时查询）
  → 点击保存
    → 成功：关闭弹窗/跳转列表，刷新列表，Toast「成员创建成功，初始密码已发送至手机」
    → 失败：表单错误提示，停留在弹窗
```

### 6.2 编辑成员

```
点击「编辑」→ 跳转 /user/:id/edit
  → 预填已有数据（密码字段留空）
  → 修改字段 → 保存
    → 成功：跳转详情页，Toast「保存成功」
    → 失败：表单错误提示
```

### 6.3 禁用/启用成员

```
点击「禁用」→ 二次确认弹窗：「确认禁用该成员？禁用后该账号将无法登录。」
  → 确认：更新状态，刷新列表，Toast「已禁用」
  → 点击「启用」同样流程，Toast「已启用」
  ⚠️ admin 不可禁用自己
  ⚠️ 不可禁用最后一个 admin 账号
```

### 6.4 重置密码

```
点击「重置密码」→ 确认弹窗：「将发送密码重置链接至 xxx@xxx.com？」
  → 确认：发送重置邮件，Toast「重置链接已发送」
  → 邮件内含24小时有效链接
```

## 7. 验收标准

1. ✅ 用户名唯一性实时校验（失焦触发，200ms防抖）
2. ✅ 密码强度校验：至少8位，包含大小写字母和数字
3. ✅ admin 不可禁用自己
4. ✅ admin 不可删除最后一个 admin
5. ✅ 禁用成员自动登出（强制下线）
6. ✅ 编辑时密码字段留空才更新，不传则保留原密码
7. ✅ 所有操作均记录到操作日志
8. ✅ 非 admin 用户访问时操作按钮不可见
9. ✅ 分页默认20条
