# 系统设置

## 1. 文档信息

- **所属系统**：AIDC Collector 管理后台
- **页面名称**：系统设置
- **路由**：
  - 个人设置：`/settings/profile`
  - 管理员设置：`/settings/admin`（仅 admin）
- **相关文档**：
  - [PRODUCT-STRUCTURE.md](./PRODUCT-STRUCTURE.md) — 整体架构
  - [COMPONENT-sidebar.md](./COMPONENT-sidebar.md) — 左边栏组件
  - [COMPONENT-form.md](./COMPONENT-form.md) — 表单组件

---

## 2. 页面定位与功能概述

系统设置模块用于管理账户个人信息和系统全局配置。分为「个人设置」（所有用户可见）和「管理员设置」（仅 admin 可见）两个子页面。

---

## 3. 用户角色与权限

| 页面 | admin | manager | viewer |
|------|-------|---------|--------|
| 个人设置 | ✅ | ✅ | ✅ |
| 管理员设置 | ✅ | ❌（不可见） | ❌（不可见） |

## 4. 功能清单

### 4.1 个人设置（所有用户）

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 头像上传 | P0 | 支持 JPG/PNG/GIF，最大 2MB |
| 姓名修改 | P0 | 真实姓名 |
| 手机号修改 | P0 | 需短信验证码 |
| 邮箱修改 | P0 | 需邮件验证码 |
| 密码修改 | P0 | 填写旧密码 + 新密码 |
| 个人偏好 | P1 | 主题（浅色/深色/跟随系统）、默认首页 |
| 通知设置 | P1 | 邮件/站内通知开关 |
| API Token 管理 | P2 | 查看/重置个人 API Token |

### 4.2 管理员设置（仅 admin）

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 系统信息 | P0 | 系统名称、Logo、版权信息 |
| 登录安全 | P0 | 登录失败锁定次数、密码策略（长度/复杂度/有效期） |
| 通知渠道 | P0 | 邮件服务器配置、企微/钉钉 Webhook |
| 数据管理 | P1 | 数据备份周期、备份保留数量 |
| 敏感操作 | P1 | 批量删除需二次确认、危险操作需 admin 二次验证 |
| 操作日志保留 | P1 | 日志保留天数（默认90天） |
| 公告管理 | P1 | 登录页/首页公告配置 |
| 系统升级 | P2 | 版本信息、检查更新 |
| 审计报告 | P2 | 定期生成安全审计报告 |

## 5. 字段定义

### 5.1 个人设置字段

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|----------|
| avatar | file | — | JPG/PNG/GIF，最大 2MB |
| name | string | ✅ | 2-30字符 |
| phone | string | ✅ | 11位手机号 |
| email | string | — | 有效邮箱格式 |
| old_password | string | 修改密码时必填 | — |
| new_password | string | 修改密码时必填 | 8-20位，大小写+数字 |
| confirm_password | string | 修改密码时必填 | 与新密码一致 |
| theme | enum | — | light / dark / system |
| default_home | string | — | 路由路径 |

### 5.2 管理员设置字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| system_name | string | ✅ | 系统名称，显示在 Logo 旁 |
| system_logo | file | — | PNG/SVG，建议尺寸 200×60px |
| copyright | string | — | 版权信息，显示在登录页底部 |
| login_max_failures | int | — | 登录失败锁定次数，默认5 |
| password_min_length | int | — | 最小密码长度，默认8 |
| password_require_uppercase | boolean | — | 必须包含大写字母 |
| password_require_number | boolean | — | 必须包含数字 |
| password_expiry_days | int | — | 密码有效期（天），0为不限制 |
| backup_enabled | boolean | — | 是否开启自动备份 |
| backup_frequency | enum | — | daily / weekly / monthly |
| backup_retention_days | int | — | 备份保留天数，默认30 |
| log_retention_days | int | — | 操作日志保留天数，默认90 |

## 6. 交互流程

### 6.1 修改密码

```
进入个人设置 → 点击「修改密码」
  → 填写：旧密码、新密码、确认密码
  → 实时校验新密码强度
  → 点击保存
    → 验证旧密码
    → 成功：更新密码，强制下线其他设备，Toast「密码修改成功」
    → 失败：提示错误原因
```

### 6.2 修改手机号

```
点击「修改手机号」
  → 填写新手机号 → 获取短信验证码
  → 填写验证码 → 提交
    → 成功：更新手机号，Toast「手机号更新成功」
    → 失败：提示验证码错误
```

### 6.3 系统 Logo 上传（admin）

```
管理员设置 → 点击「更换 Logo」
  → 上传图片文件（PNG/SVG，最大 2MB）
  → 实时预览效果
  → 点击保存 → 更新全站 Logo
```

### 6.4 密码策略修改（admin）

```
管理员设置 → 登录安全
  → 修改策略 → 保存
  → 提示：「密码策略修改后，新注册用户和修改密码时生效，现有用户不受影响」
```

---

## 7. 验收标准

1. ✅ 个人设置修改无需审批，立即生效
2. ✅ 修改手机号/邮箱必须验证后才生效
3. ✅ 修改密码后强制当前设备重新登录
4. ✅ 密码强度实时可视化（弱/中/强三档）
5. ✅ admin 修改密码策略时有二次确认
6. ✅ 所有设置修改记录到操作日志
7. ✅ 系统设置（admin）修改前可选择「发布公告」通知所有用户
8. ✅ Logo/名称修改后全站即时生效（无需重启）
9. ✅ 非 admin 用户访问 /settings/admin 跳转404或无权限页
