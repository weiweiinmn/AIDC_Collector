## 6. 排序规则（新增）

### 6.1 项目排序

| 排序维度 | 规则 | 说明 |
|----------|------|------|
| 默认 | `createdAt` 倒序 | 最新创建的项目在前 |
| 可选（未来） | 按名称字母序 | A-Z |
| 可选（未来） | 按功率大小 | 大功率优先 |

### 6.2 任务排序（单项目内）

| 排序维度 | 规则 | 说明 |
|----------|------|------|
| 默认 | `stage` 顺序 + `start` 正序 | 按阶段（签约→算力），同阶段按开始日期 |
| 阶段顺序 | signing → delivery → boi → purchase → entry → compute | 固定生命周期顺序 |

**当前实现**：固定按阶段生命周期顺序排列，不可拖拽排序。

## 7. 数据上限与性能边界（新增）

### 7.1 上限定义

| 维度 | 上限 | 超限处理 |
|------|------|----------|
| 项目数 | 100 个 | 超出时提示「项目数量过多，建议归档已完成项目」 |
| 单项目任务数 | 50 个 | 超出时「+ 任务」按钮 disabled，tooltip 提示 |
| 任务名称长度 | 100 字符 | 输入框 maxlength，超出截断 |
| 备注长度 | 500 字符 | 同上 |

### 7.2 性能边界

| 场景 | 目标性能 | 优化策略 |
|------|----------|----------|
| 初始渲染 | < 200ms（50项目×10任务） | 虚拟滚动（未来）、减少重绘 |
| 筛选切换 | < 50ms | 仅重渲染可见行 |
| 展开/折叠 | < 30ms | 局部 DOM 更新 |
| 弹窗打开 | < 100ms | 预渲染、减少计算 |

### 7.3 大数据提示

当 `projects.length > 20` 或总任务数 > 200 时：
- ctrl-bar 显示提示：「数据量较大，建议使用筛选功能」
- 默认折叠所有项目，仅展开最近更新的 3 个

## 8. 错误处理与异常场景（新增）

### 8.1 表单校验错误

| 场景 | 错误提示 | 处理方式 |
|------|----------|----------|
| 项目名称为空 | 「请输入项目名称」 | input 边框变红，聚焦 |
| 任务名称为空 | 「请输入任务名称」 | 同上 |
| 结束日期 < 开始日期 | 「结束日期不能早于开始日期」 | alert 提示，不关闭弹窗 |
| 进度非数字 | 「进度必须是 0-100 的数字」 | input 边框变红 |

### 8.2 存储错误

| 场景 | 错误提示 | 处理方式 |
|------|----------|----------|
| localStorage 写入失败 | 「数据保存失败，请检查浏览器存储权限」 | console.error，不阻断操作 |
| localStorage 满 | 「存储空间不足，请清理浏览器数据」 | 提示用户，数据仅内存保留 |
| 数据解析失败 | 「数据损坏，已重置为默认数据」 | 清空 storage，加载 mock |

### 8.3 关联数据缺失

| 场景 | 处理方式 |
|------|----------|
| 关联机房被删除 | 项目弹窗显示「关联机房已不存在」，dcId 保留但显示为灰色 |
| 机房列表为空 | 下拉框显示「暂无机房数据，请先添加机房」 |

### 9.1 当前架构（纯前端）

```
│   gantt.js  │ ←→  │  API Layer  │ ←→  │   Backend   │
└─────────────┘     └─────────────┘     └─────────────┘
                          ↓
                    ┌─────────────┐
                    │ localStorage│ (离线缓存)
                    └─────────────┘
```

### 9.3 API 预留接口

```javascript
// 当前：直接操作 localStorage
function saveProjects() {
    localStorage.setItem(STORAGE_KEY, JSON.stringify(projects));
}

// 未来：API 层封装
var API = {
    // 项目
    getProjects: () => fetch('/api/gantt/projects').then(r => r.json()),
    createProject: (data) => fetch('/api/gantt/projects', {method:'POST', body:JSON.stringify(data)}),
    updateProject: (id, data) => fetch(`/api/gantt/projects/${id}`, {method:'PUT', body:JSON.stringify(data)}),
    deleteProject: (id) => fetch(`/api/gantt/projects/${id}`, {method:'DELETE'}),
    
    // 任务
    getTasks: (projId) => fetch(`/api/gantt/projects/${projId}/tasks`).then(r => r.json()),
    createTask: (projId, data) => fetch(`/api/gantt/projects/${projId}/tasks`, {method:'POST', body:JSON.stringify(data)}),
    updateTask: (projId, taskId, data) => fetch(`/api/gantt/projects/${projId}/tasks/${taskId}`, {method:'PUT', body:JSON.stringify(data)}),
    deleteTask: (projId, taskId) => fetch(`/api/gantt/projects/${projId}/tasks/${taskId}`, {method:'DELETE'}),
};

// 数据同步策略
async function syncProjects() {
    // 1. 尝试从 API 获取
    try {
        var remote = await API.getProjects();
        projects = remote;
        localStorage.setItem(STORAGE_KEY, JSON.stringify(projects)); // 缓存
    } catch (e) {
        // 2. 失败则从缓存读取
        projects = loadProjects();
    }
    renderGantt();
}
```

## 12. 甘特条交互细节（新增）

### 12.1 鼠标交互

| 操作 | 行为 | 视觉反馈 |
|------|------|----------|
| **Hover** | 显示 Tooltip | 条形抬起 `translateY(-1px)` + 阴影加深 |
| **单击** | 打开任务编辑弹窗 | 无额外反馈（弹窗打开即反馈） |
| **双击** | 同单击（不区分） | - |
| **右键** | 浏览器默认菜单（无自定义右键菜单） | - |

### 12.2 Tooltip 规格

**触发条件**：鼠标在甘特条上停留 **300ms** 后显示

**位置**：条形上方居中，距离条形顶部 **8px**

**内容格式**：
```
┌─────────────────────────────────────┐
│ 合同签署                            │  ← 任务名称，14px，白色
│ 2026-01-15 → 2026-03-20 (65天)     │  ← 日期范围，12px，灰色
│ 进度：65%                           │  ← 进度，12px，阶段色
│ 备注：等待法务审核                  │  ← 备注（如有），11px，灰色，最多2行
└─────────────────────────────────────┘
```

**样式**：
- 背景：`rgba(0,0,0,.9)`
- 圆角：6px
- 内边距：10px 14px
- 最大宽度：280px
- 阴影：`0 4px 20px rgba(0,0,0,.4)`

**消失条件**：鼠标离开条形或停留 **5秒** 自动消失

### 12.3 空日期任务

任务 `start` 或 `end` 为空时：
- **不渲染甘特条**
- **左侧任务名后显示标签**：`[未排期]`，11px，橙色 `#F59E0B`
- **点击任务行**：仍可打开编辑弹窗设置日期

### 13.1 日期输入控件

**组件**：原生 `<input type="date">` + 快捷按钮组

**布局**：
```
┌─────────────────────────────────────────┐
```

**快捷按钮**：

|------|------|----------|
| **今天** | 填充今天日期 | 任务从今天开始 |
| **明天** | 填充明天日期 | 任务明天开始 |
| **一周后** | 填充 `今天+7天` | 常用工期起点 |

**日期校验**：
- 结束日期 ≥ 开始日期（允许相等，表示单日任务）

**双模式设计**：

**模式 A：滑块（默认）**

- 范围：0-100
- 步进：5（拖动时）
- 点击刻度直接跳转

**模式 B：数字输入**

- 失焦时自动校验范围，超限取边界值

**切换方式**：滑块右侧显示数字输入框，双向同步

## 15. Undo/Redo 与数据安全（新增）

### 15.1 操作历史

**记录范围**：
- ✅ 任务删除
- ✅ 项目删除
- ✅ 任务批量修改（未来）

**不记录**：
- ❌ 筛选切换
- ❌ 展开/折叠
- ❌ 日期导航

**实现方式**：
```javascript
var historyStack = [];  // 最多保存 10 步
var historyIndex = -1;

// 删除前保存快照
function deleteTask(projId, taskId) {
    var snapshot = JSON.parse(JSON.stringify(projects));
    historyStack.push(snapshot);
    if (historyStack.length > 10) historyStack.shift();
    
    // 执行删除
    // ...
}

// Undo
function undo() {
    if (historyStack.length === 0) return;
    projects = historyStack.pop();
    saveProjects();
    renderGantt();
    showToast('已撤销删除');
}
```

### 15.2 删除确认

| 操作 | 确认方式 | 文案 |
|------|----------|------|
| 删除任务 | `confirm()` 弹窗 | 「确定删除任务「XXX」？此操作不可撤销。」 |
| 删除项目 | `confirm()` 弹窗 | 「确定删除项目「XXX」及其 N 个任务？此操作不可撤销。」 |

### 15.3 操作反馈

| 操作 | 反馈方式 | 文案/效果 |
|------|----------|-----------|
| 保存成功 | Toast，2秒消失 | 「已保存」 |
| 删除成功 | Toast，2秒消失 | 「已删除」+ Undo 按钮 |
| 删除撤销 | Toast，2秒消失 | 「已撤销删除」 |
| 保存失败 | Alert | 「保存失败，请重试」 |

### 15.4 自动保存草稿

**弹窗内自动保存**：
- 触发：字段失焦（blur）后 500ms
- 存储：`sessionStorage.setItem('gantt_draft_' + modalType, JSON.stringify(formData))`
- 恢复：弹窗打开时读取 sessionStorage，如有草稿提示「是否恢复上次未保存的内容？」
- 清理：成功保存后清除草稿

### 17.1 Excel 导入

**入口**：ctrl-bar 「导入」按钮（与导出并列）

**流程**：
1. 点击「导入」→ 打开文件选择器（accept=".xlsx,.xls"）
2. 前端解析 Excel（使用 SheetJS）
3. 预览确认（显示前 5 行）
4. 确认后合并/替换现有数据

**格式要求**：
| 列 | 字段 | 必填 | 示例 |

## 19. 并发编辑与数据冲突（新增）

### 19.1 当前限制

**localStorage 单用户模式**：
- 不支持多用户同时编辑
- 不支持跨设备实时同步
- 同一浏览器多标签页：最后保存的覆盖前者

### 19.2 冲突检测（未来 API 化）

```javascript
// 保存时检测版本
async function saveWithConflictCheck(project) {
    var remote = await API.getProject(project.id);
    if (remote.version !== project.version) {
        // 冲突！
        showConflictResolver(local, remote);
    }
}
```

### 19.3 用户提示

当检测到可能的多标签页编辑时：
- 保存前读取 localStorage 对比当前数据哈希
- 如不一致，提示「数据可能已被其他页面修改，是否覆盖？」
